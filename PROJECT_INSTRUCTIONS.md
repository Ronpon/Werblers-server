# Project Instructions for AI Assistants

You must follow these rules when generating or modifying code:

1. Always read VISION.md and RULES.md before making changes.
2. Never invent rules that are not explicitly defined in RULES.md.
3. If any required behavior is not defined in RULES.md:
   - Do NOT assume.
   - Do NOT guess.
   - Instead, stop and list the missing rule.
4. If you make an assumption for implementation simplicity, you must:
   - Explicitly list it under a section called "Assumptions Made".
   - Justify why it was necessary.
5. Do not implement features listed as "Out of Scope".
6. Keep architecture minimal.
7. Prefer simple procedural logic over abstractions.
8. If unclear, ask questions before writing code.
